Introduction

Cargo is a high-performance, S3-compatible object storage gateway designed for Hack Clubbers. It provides a simple, familiar interface for storing and retrieving files, backed by robust infrastructure.

Whether you're hosting assets for a game, storing user uploads, or backing up data, Cargo offers a standard S3 API that works with your favorite tools and libraries.

Authentication

To access Cargo programmatically, you need an Access Key and a Secret Key.

  1. Log in to the Dashboard using your Hack Club Slack account.
  2. Create a new Bucket.
  3. Click on the bucket to view its details.
  4. Generate a new Access Key pair.

Important: Your Secret Key is only shown once. Make sure to save it securely. If you lose it, you'll need to generate a new pair.

Public Buckets

By default, all buckets are private. This means every request requires valid authentication signatures.

You can toggle a bucket to be Public in the dashboard settings.

What does Public mean?

  • Anyone can perform GetObject and HeadObject requests without authentication.
  • Files are accessible via direct URL: https://cargo.deployor.dev/bucket-name/key.
  • ListObjects and other operations still require authentication.

This is ideal for hosting static assets like images, CSS, or game files that need to be publicly accessible on the web.

Supported Operations

Cargo implements a subset of the Amazon S3 API. The following operations are fully supported:

Bucket Operations

  • ✓ ListBuckets (GET /)
  • ✓ HeadBucket (HEAD /bucket)
  • ✓ GetBucketLocation (GET /bucket?location)
  • ✗ CreateBucket (Use Dashboard)
  • ✗ DeleteBucket (Use Dashboard)
  • ✗ Put/GetBucketAcl
  • ✗ Put/GetBucketCors
  • ✗ Put/GetBucketPolicy
  • ✗ Put/GetBucketVersioning
  • ✗ Put/GetBucketTagging
  • ✗ Put/GetBucketLifecycle
  • ✗ Put/GetBucketReplication
  • ✗ Put/GetBucketEncryption
  • ✗ Put/GetBucketWebsite
  • ✗ Put/GetPublicAccessBlock
  • ✗ ListObjects (v1)
  • ✗ Analytics/Inventory/Metrics

Object Operations

  • ✓ PutObject (PUT /bucket/key)
  • ✓ GetObject (GET /bucket/key)
  • ✓ HeadObject (HEAD /bucket/key)
  • ✓ DeleteObject (DELETE /bucket/key)
  • ✓ CopyObject (PUT /bucket/key + x-amz-copy-source)
  • ✓ ListObjectsV2 (GET /bucket?list-type=2)
  • ✓ DeleteObjects (POST /bucket?delete)
  • ✗ Put/GetObjectAcl
  • ✗ Put/GetObjectTagging
  • ✗ Put/GetObjectLegalHold
  • ✗ Put/GetObjectRetention
  • ✗ Put/GetObjectLockConfig
  • ✗ RestoreObject
  • ✗ SelectObjectContent

Multipart Uploads

Fully supported for large files.

  • ✓ CreateMultipartUpload (POST /bucket/key?uploads)
  • ✓ UploadPart (PUT /bucket/key?partNumber&uploadId)
  • ✓ CompleteMultipartUpload (POST /bucket/key?uploadId)
  • ✓ AbortMultipartUpload (DELETE /bucket/key?uploadId)
  • ✓ ListMultipartUploads (GET /bucket?uploads)
  • ✓ ListParts (GET /bucket/key?uploadId)

Endpoints & Regions

Region

eu-central-1

Endpoint URL

https://cargo.deployor.dev

Addressing Styles

We support both virtual-hosted-style and path-style access.

// Virtual Host (Recommended) https://{bucket}.cargo.deployor.dev/{key}
// Path Style https://cargo.deployor.dev/{bucket}/{key}

Limits & Quotas

Storage & Bandwidth

We monitor storage usage and egress bandwidth.

  • Storage Limit: Defined per user (check dashboard).
  • Egress Limit: Typically 3x your storage limit.

Unsupported Features

The following features are currently not implemented or blocked. Requests containing these headers or parameters will return a 501 Not Implemented error.

ACLs Bucket Policy Encryption Versioning Lifecycle Replication Object Lock

AWS CLI

You can use the standard AWS CLI to interact with Cargo. Configure a profile with your keys and our endpoint.

Configuration

aws configure --profile cargo
# AWS Access Key ID: [Your Access Key]
# AWS Secret Access Key: [Your Secret Key]
# Default region name: eu-central-1
# Default output format: json

Usage Examples

# List Buckets
aws s3 ls --endpoint-url https://cargo.deployor.dev --profile cargo

# Upload a file
aws s3 cp myfile.txt s3://my-bucket/myfile.txt --endpoint-url https://cargo.deployor.dev --profile cargo

# List Objects
aws s3 ls s3://my-bucket --endpoint-url https://cargo.deployor.dev --profile cargo

JavaScript / Bun

Use the @aws-sdk/client-s3 package. 

import { S3Client, PutObjectCommand } from "@aws-sdk/client-s3";

const s3 = new S3Client({
  region: "eu-central-1",
  endpoint: "https://cargo.deployor.dev",
  credentials: {
    accessKeyId: "YOUR_ACCESS_KEY",
    secretAccessKey: "YOUR_SECRET_KEY",
  },
});

await s3.send(
  new PutObjectCommand({
    Bucket: "my-bucket",
    Key: "hello.txt",
    Body: "Hello World!",
  })
);
